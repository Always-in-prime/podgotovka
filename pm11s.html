1. Универсальный шаблон проектирования БД

Любая предметная область строится из 4 типов сущностей. Определите их – и вы уже на полпути.

👤 Люди Сотрудники, клиенты, студенты, жильцы. Поля: ID, ФИО, дата рождения, пол, телефон, адрес.
🏢 Объекты Станки, комнаты, товары, дома. Поля: ID, название, характеристики (цена, площадь).
🔗 Связи Заказы, бронирования, ремонты, проживание. Поля: ID_объекта, ID_человека, дата, сумма.
👶 Дочерние Дети, замки, компьютеры – часто отдельные таблицы или атрибуты.
Запомните: 3–4 таблицы + внешние ключи + поля для фильтрации (даты, статусы, суммы).

2. Шаблоны SQL-запросов – 6 типов

Нажмите на любой шаблон – появится пример из реальной задачи.

1. Простая фильтрация (WHERE)
SELECT * FROM Таблица
WHERE поле = 'значение'
AND дата > '2020-01-01'
-- Пример: сотрудники, принятые после 01.02.2012, фамилия на 'Б'
SELECT * FROM Сотрудники
WHERE Дата_приёма > '2012-02-01'
AND ФИО LIKE 'Б%'
2. JOIN – данные из двух таблиц
SELECT t1.поле, t2.поле
FROM Таблица1 t1
JOIN Таблица2 t2 ON t1.ID = t2.ID_Связи
WHERE ...
-- Пример: клиент, купивший колонки SVEN по минимальной цене
SELECT Клиенты.ФИО
FROM Клиенты
JOIN Заказы ON Клиенты.ID = Заказы.ID_Клиента
JOIN ДеталиЗаказа ON Заказы.ID = ДеталиЗаказа.ID_Заказа
JOIN Товары ON ДеталиЗаказа.ID_Товара = Товары.ID
WHERE Товары.Название = 'SVEN'
AND Товары.Цена = (SELECT MIN(Цена) FROM Товары WHERE Название='SVEN')
3. Группировка (GROUP BY)
SELECT поле_группы, COUNT(*), SUM(поле)
FROM ...
GROUP BY поле_группы
HAVING COUNT(*) > N
-- Пример: сводка по полу детей в отделах
SELECT Отдел,
SUM(CASE WHEN Дети.Пол='М' THEN 1 ELSE 0 END) AS Мальчики,
SUM(CASE WHEN Дети.Пол='Ж' THEN 1 ELSE 0 END) AS Девочки
FROM Сотрудники
JOIN Дети ON Сотрудники.ID = Дети.ID_Сотр
GROUP BY Отдел
4. Подзапросы (MIN/MAX, EXISTS)
SELECT * FROM Таблица
WHERE поле = (SELECT MAX(поле) FROM ...)
-- или EXISTS
WHERE EXISTS (SELECT 1 FROM ...)
-- Пример: сотрудники, имеющие детей (EXISTS)
SELECT * FROM Сотрудники
WHERE EXISTS (SELECT 1 FROM Дети WHERE Дети.ID_Сотр = Сотрудники.ID)
5. Сортировка (ORDER BY)
SELECT * FROM Таблица
ORDER BY поле ASC -- от А до Я
ORDER BY поле DESC -- от Я до А
-- Пример: физ. лица, сортировка по фамилии от Я до А
SELECT ФИО FROM Клиенты
WHERE ТипЛица = 'физ'
ORDER BY ФИО DESC
6. Фильтрация по датам и тексту
WHERE дата > '2013-04-01'
WHERE фамилия LIKE 'Б%'
WHERE цена BETWEEN 200 AND 500
-- Пример: товары дороже 200 грн, купленные после 01.04.2013
SELECT DISTINCT Товары.Название, Товары.Цена
FROM Товары
JOIN ДеталиЗаказа ON Товары.ID = ДеталиЗаказа.ID_Товара
JOIN Заказы ON ДеталиЗаказа.ID_Заказа = Заказы.ID
WHERE Товары.Цена > 200 AND Заказы.Дата > '2013-04-01'

3. Ключевые фразы → действие (кликабельно)

Фраза в заданииЧто делать
«Список …»SELECT * FROM + WHERE
«Сводные данные по …»GROUP BY + агрегаты
«Общее количество …»SELECT COUNT(*)
«Средний …»SELECT AVG(поле)
«Минимальная/максимальная цена»подзапрос с MIN/MAX
«Клиент, купивший …»JOIN Заказы + Товары
«Сотрудники, имеющие детей»EXISTS
«Отсортировать от Я до А»ORDER BY поле DESC
«После такой-то даты»WHERE дата > 'дата'
«В диапазоне»BETWEEN ... AND ...
«Фамилия начинается на …»LIKE 'Б%'
«Пенсионеры, призывники, дети до 7 лет»DATEDIFF(...) >= N

4. Шаблоны для программных задач

🔢 Високосный год
if (год % 400 == 0) or (год % 4 == 0 and год % 100 != 0):
    дни = 366
else:
    дни = 365
💵 Склонение «рубль» / «копейка»
Проверяем n % 100. Если 11–19 → родительный падеж. Иначе по последней цифре: 1 → ед.ч., 2–4 → род. падеж ед.ч., 0,5–9 → род. падеж мн.ч.
➖ Пример на вычитание
Генерируем a=rand(1,100), b=rand(1,100), выводим a-b=?, ждём ответ, сравниваем.
🔁 Перевод в двоичную
Делим на 2, записываем остатки, переворачиваем. Либо bin(n)[2:].

5. Мнемонический каркас – на один лист

1. Таблицы: Люди, Объекты, Связи, Дочерние.
2. SELECT ... FROM ... JOIN ... ON ... WHERE ... GROUP BY ... HAVING ... ORDER BY ...
3. Агрегаты: COUNT, SUM, AVG, MIN, MAX
4. Подзапросы: = (SELECT ...), EXISTS (SELECT 1 ...)
5. Даты: > 'ГГГГ-ММ-ДД', BETWEEN
6. Текст: LIKE '...%', '%...', '...%...'

6. Пошаговый алгоритм решения любой задачи

  1. Прочитайте условие и выделите существительные – это таблицы.
  2. Определите, какие поля нужны в выводе – их берём в SELECT.
  3. Есть ли связь между таблицами? → JOIN.
  4. Есть ли группировка? → GROUP BY.
  5. Есть ли фильтр? → WHERE.
  6. Есть ли сортировка? → ORDER BY.
  7. Напишите запрос по шаблону, подставляя имена полей.

7. Пример – разбор задачи

Задача: «Вывести список всех трёхкомнатных квартир в доме №5 на улице Ленина».

  • → Таблицы: Квартиры, Дома, Улицы
  • → SELECT Квартиры.* FROM Квартиры
  • → JOIN Дома ON Квартиры.ID_Дома = Дома.ID
  • → JOIN Улицы ON Дома.ID_Улицы = Улицы.ID
  • → WHERE Улицы.Название = 'Ленина' AND Дома.Номер = 5 AND Квартиры.Комнат = 3
