Как легко «щелкать» любые задачи на экзамене – только шаблоны

1. Универсальный шаблон проектирования БД

Любая предметная область (завод, университет, жилконтора, гостиница, страховая) строится из одних и тех же типов сущностей:

👤 Люди Сотрудники, рабочие, клиенты, студенты, жильцы. Поля: ID, ФИО, дата рождения, пол, телефон, адрес, должность.
🏢 Объекты Станки, помещения, комнаты, товары, дома. Поля: ID, название, характеристики (цена, площадь, мощность).
🔗 Связи Заказы, бронирования, страховки, ремонты, проживание. Поля: ID_объекта, ID_человека, дата, сумма, количество.
👶 Дочерние Дети, замки, компьютеры – часто отдельные таблицы или атрибуты.
Запомните: 3–4 главные таблицы + внешние ключи + поля для фильтрации (даты, суммы, статусы).

2. Шаблоны SQL-запросов – 6 типов

Все задачи сводятся к этим конструкциям. Подставляйте свои таблицы и поля.

2.1. Простая фильтрация (WHERE)

SELECT * FROM Таблица WHERE поле = 'значение' AND поле > 100 AND дата > '2020-01-01'

Ключевые слова «список», «вывести», «кто», «какие» + ограничения.

2.2. JOIN – данные из двух таблиц

SELECT Таблица1.поле, Таблица2.поле FROM Таблица1 JOIN Таблица2 ON Таблица1.ID = Таблица2.ID_Связи WHERE ...

Ключевые слова «клиент, купивший», «сотрудники, имеющие детей», «заказы с товарами».

2.3. Группировка и агрегаты (GROUP BY)

SELECT поле_группы, COUNT(*), SUM(поле), AVG(поле) FROM ... GROUP BY поле_группы HAVING COUNT(*) > ...

Ключевые слова «сводные данные», «общее количество», «средний», «суммарный», «по каждому отделу/этажу/кафедре».

2.4. Подзапросы (вложенные SELECT)

SELECT * FROM Таблица WHERE поле = (SELECT MAX(поле) FROM ...) -- или EXISTS WHERE EXISTS (SELECT 1 FROM ... WHERE ...)

Ключевые слова «минимальная цена», «максимальный оклад», «те, кто имеют детей» (проверка существования).

2.5. Сортировка (ORDER BY)

SELECT * FROM Таблица ORDER BY поле ASC -- от А до Я, по возрастанию ORDER BY поле DESC -- от Я до А, по убыванию

Ключевые слова «отсортировать», «по убыванию/возрастанию».

2.6. Фильтрация по датам и тексту

WHERE дата > '2013-04-01' WHERE фамилия LIKE 'Б%' -- начинается на Б WHERE фамилия LIKE '%ов' -- заканчивается на "ов" WHERE фамилия LIKE 'С%В' -- начинается на С и заканчивается на В WHERE цена BETWEEN 200 AND 500

Ключевые слова «после даты», «в диапазоне», «фамилия начинается/заканчивается».

3. Как перевести условие в SQL – по ключевым фразам

Фраза в заданииЧто делать
«Список …»SELECT * FROM нужная таблица + WHERE
«Сводные данные по …»GROUP BY + агрегаты (COUNT, SUM)
«Общее количество …»SELECT COUNT(*)
«Средний …»SELECT AVG(поле)
«Минимальная/максимальная цена»подзапрос с MIN/MAX
«Клиент, купивший …»JOIN Заказы + Товары + WHERE
«Сотрудники, имеющие детей»EXISTS (подзапрос к Детям)
«Отсортировать от Я до А»ORDER BY поле DESC
«После такой-то даты»WHERE дата > 'дата'
«В диапазоне»BETWEEN ... AND ...
«Фамилия начинается на …»LIKE 'Б%'
«Пенсионеры, призывники, дети до 7 лет»DATEDIFF(year, дата_рождения, GETDATE()) >= N

4. Шаблоны для программных задач (№42,44,46,48,50)

🔢 Високосный год
if (год % 400 == 0) or (год % 4 == 0 and год % 100 != 0): дни = 366 else: дни = 365
💵 Склонение «рубль» / «копейка»
Берём n % 100. Если 11–19 → родительный падеж. Иначе по последней цифре: 1 → ед.ч., 2–4 → род.падеж ед.ч., 0,5–9 → род.падеж мн.ч.
➖ Пример на вычитание
Генерируем a=rand(1,100), b=rand(1,100), выводим a-b=?, ждём ответ, сравниваем.
🔁 Перевод в двоичную
Делим число на 2, записываем остатки, переворачиваем. Либо bin(n)[2:].

5. Мнемонический каркас – на один лист

1. Таблицы: Люди, Объекты, Связи, Дочерние.
2. SELECT ... FROM ... JOIN ... ON ... WHERE ... GROUP BY ... HAVING ... ORDER BY ...
3. Агрегаты: COUNT, SUM, AVG, MIN, MAX
4. Подзапросы: = (SELECT ...), EXISTS (SELECT 1 ...)
5. Даты: > 'ГГГГ-ММ-ДД', BETWEEN
6. Текст: LIKE '...%', '%...', '...%...'

6. Пошаговый алгоритм решения любой задачи (за 3 минуты)

  1. Прочитайте условие и выделите существительные – это таблицы.
  2. Определите, какие поля нужны в выводе – их берём в SELECT.
  3. Есть ли связь между таблицами? → JOIN.
  4. Есть ли группировка? → GROUP BY.
  5. Есть ли фильтр? → WHERE.
  6. Есть ли сортировка? → ORDER BY.
  7. Напишите запрос по шаблону, подставляя имена полей.

7. Пример – разбор задачи

Задача: «Вывести список всех трёхкомнатных квартир в доме №5 на улице Ленина».

  • → Таблицы: Квартиры, Дома, Улицы
  • → SELECT Квартиры.* FROM Квартиры
  • → JOIN Дома ON Квартиры.ID_Дома = Дома.ID
  • → JOIN Улицы ON Дома.ID_Улицы = Улицы.ID
  • → WHERE Улицы.Название = 'Ленина' AND Дома.Номер = 5 AND Квартиры.Комнат = 3
